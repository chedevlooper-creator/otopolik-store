# Milestone v1.1: AI Destekli Lüks Deneyim

**Status:** ✅ SHIPPED 2026-07-17
**Phases:** 5-8
**Total Plans:** 12 (across 4 phases)

## Overview

Layer AI capabilities over the shipped v1.0 luxury storefront — customer-facing assistants for vehicle matching, configuration, and support, plus an admin content generator — without touching the core Convex commerce backbone. Every price/policy claim traces back to code (`mat-pricing.ts`, `getVehiclePrice()`, site-settings), never model-composed. AI is an additive layer: with keys unset or `AI_FEATURES_ENABLED=false`, the storefront runs entirely on its existing manual flows.

## Phases

### Phase 5: AI Infrastructure & Vehicle Matcher

**Goal**: Customers can type messy Turkish vehicle text and land on the correct brand/model/price tier, while every AI surface shares a safe server-only client, cost controls, and an eval harness
**Depends on**: Nothing (first v1.1 phase; v1.0 shipped)
**Requirements**: AIINF-01..06, VMATCH-01..04
**Plans**: 3 plans

Plans:

- [x] 05-01: Shared AI client, cost controls, customer-only price tools
- [x] 05-02: Deterministic-first vehicle match pipeline + `/api/ai/*` routes
- [x] 05-03: VehicleMatchInput UI, MatConfigurator prefill, golden eval harness

**Details:**
Server-only AI client layer under `src/app/api/ai/*` keeps the provider key out of the browser. Deterministic fuzzy/token search (`vehicle-search.ts`) runs first; the LLM only parses messy text or disambiguates 0-/multi-match cases. Ambiguous input yields a candidate disambiguation list — never a silent wrong guess. A resolved vehicle pre-fills the existing `MatConfigurator` step. Cost controls (rate limit / token caps / kill switch) and a golden eval harness (price-equality + Turkish quality assertions) stand up here for all later phases to reuse.

### Phase 6: Streaming Chat & Configurator Assistant

**Goal**: Customers can complete a full mat configuration via a premium Turkish streaming AI chat that drives the real stepper and adds a correctly priced item to the real cart
**Depends on**: Phase 5
**Requirements**: CFGAI-01..05
**Plans**: 3 plans

Plans:

- [x] 06-01: Streaming `/api/ai/chat` + configurator tools (reuse Phase 5 client/rate-limit/prices)
- [x] 06-02: Premium ConfiguratorChat drives real MatConfigurator + cart + WhatsApp
- [x] 06-03: Price-equality golden evals + stream reconnect polish + human verify

**Details:**
A Turkish streaming chat walks vehicle → floor/edge colors → extras → price → WhatsApp handoff. Chat actions drive the existing `MatConfigurator` stepper via tool calls (no parallel/fictional checkout) and add to the real cart via `useCart().addItem()` with prices equal to `calculateMatPrice`. UI matches the OLED-black/glass premium system, mobile-first, and is clearly labeled "AI Asistan".

### Phase 7: Support / Order Helper & Grounding

**Goal**: Customers can get shipping/sizing/care answers grounded in live site content and draft a WhatsApp order message without the AI placing the order
**Depends on**: Phase 6
**Requirements**: SUPAI-01..05
**Plans**: 3 plans

Plans:

- [x] 07-01: Live CMS reads + support grounding tools + streaming `/api/ai/support`
- [x] 07-02: Premium `/destek` SupportChat + user-send WhatsApp drafts
- [x] 07-03: Golden evals pass; human provider-on verification

**Details:**
Support answers are grounded in live CMS/site-settings re-fetched at answer time (this phase also fixed `cms.ts` `getFaqs`/`getContentPage` to read Convex instead of static fallbacks). Off-topic questions get a scoped Turkish redirect. Uncertain cases produce a pre-filled `wa.me` handoff; the assistant drafts an order summary that the user — not the AI — sends. Transcripts follow KVKK data-minimization (short-TTL/stateless, no training pipeline).

### Phase 8: Admin Content Generator

**Goal**: Admins can generate premium Turkish product/SEO/FAQ copy as drafts and publish only after explicit review
**Depends on**: Phase 5 (independent of customer chat)
**Requirements**: CNTGEN-01..04
**Plans**: 3 plans

Plans:

- [x] 08-01: Draft schema, style guide, product/vehicle grounding
- [x] 08-02: Admin generate route, publish action, ContentManager AI Taslak UI
- [x] 08-03: Golden evals pass; human verify on /admin/icerik

**Details:**
Admin-only generator produces Turkish product descriptions, SEO titles/meta, and FAQ copy grounded in structured product/vehicle facts. Content lands as an isolated `contentGenerations` draft and never goes live until an explicit authenticated publish action. The generator lives inside `ContentManager.tsx` (`/admin/icerik`), is admin-key-gated, and follows a written premium ("Apple/Porsche") Turkish brand-voice style guide. The "AI Taslak" tab hides entirely when AI is not configured.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- AI provider keys are server-only — all customer AI calls go through `src/app/api/ai/*` route handlers; admin generation runs as a Convex `"use node"` internalAction.
- Graceful degradation: with `AI_FEATURES_ENABLED`/keys unset, every AI surface falls back to the pre-existing manual flow (mirrors the Convex-first + static-fallback pattern).
- Prices/policies are never free-generated — always sourced from `calculateMatPrice` / `getVehiclePrice` / site-settings via tools; enforced by a price-equality golden eval.
- Vehicle matcher is deterministic-first (fuzzy/token) with LLM fallback only for messy/ambiguous input.
- AI never places orders — it drafts WhatsApp messages; the customer presses send (preserves the WhatsApp-confirmed trust model).
- Support answers are grounded in live CMS/settings at answer time and refuse off-topic queries.
- Admin content is draft-then-publish only and structurally isolated from customer chat tool surfaces.

**Issues Resolved:**

- `cms.ts` `getFaqs`/`getContentPage` returned static fallbacks instead of live Convex data — fixed in Phase 7 so support grounding reflects real CMS edits.

**Issues Deferred:**

- Conversation analytics (which questions the AI can't answer / abandonment points) — future v1.x after validation.
- Proactive contextual assistant nudges on `/arac/[slug]` pages — future v1.x.
- Multi-language content generation — only if non-Turkish markets are served.

**Technical Debt Incurred:**

- Pre-existing React hydration warning on `/olusturucu` (present before Phase 6; not a regression) — documented in 06-VERIFICATION.md.
- AI-on runtime paths were verified via golden evals rather than live provider verification (keys unset in the dev environment); live provider UX verification recommended before production AI enablement.
