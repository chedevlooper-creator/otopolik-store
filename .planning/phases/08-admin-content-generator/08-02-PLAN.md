<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/admin/ai/content | Must require admin_session; AI key never returned to client |
| Browser → Convex draft queries | adminKey only via /api/admin/convex-key after session check |
| Generate → live CMS | Must not cross; only publish action may write live tables |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-04 | Spoofing | /api/admin/ai/content | high | mitigate | isAuthenticated before any generation; 401 otherwise |
| T-08-05 | Elevation of Privilege | publishContentGenerationAction | high | mitigate | guard() + getAdminConvexKey + requireAdminKey on Convex writes |
| T-08-06 | Denial of Service | generate route | medium | mitigate | checkAiRateLimit + token caps from config |
| T-08-07 | Tampering | Auto-publish bypass | critical | mitigate | Separate modules/actions; generate route forbidden from calling live CMS saves |
| T-08-08 | Information Disclosure | adminKey in browser | medium | mitigate | Only via existing session-gated convex-key route; never expose ANTHROPIC_API_KEY |
| T-08-SC | Tampering | npm installs | high | mitigate | No new packages required — reuse ai / @ai-sdk/anthropic already present |
</threat_model>
