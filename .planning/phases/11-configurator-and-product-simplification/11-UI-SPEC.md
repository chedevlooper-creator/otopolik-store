# Phase 11 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase framing:** Phase 11 simplifies three existing surfaces — `MatConfigurator.tsx` (stepper,
color merge, extras, mobile sticky bar), `/urunler` listing chrome, and `/urunler/[slug]` detail
page — by reducing decision surfaces and motion, not by introducing a new visual language. All
13 decisions in `11-CONTEXT.md` (D-01…D-13) are locked; this contract translates them into
concrete spacing/typography/color/copy so planner and executor need zero design judgment calls.
No shadcn, no new dependencies, no new CSS utility classes — everything reuses `globals.css`
(`btn-red-rich`, `btn-press`, `surface-glass`, `spec-value`, `spec-label`) and existing components
(`ColorPicker`, `ConfigSummary`, `ExtrasSelector`, `ProductCard`).

## Spacing Scale

Declared values (multiples of 4) — inherited unchanged from the existing Tailwind v4 scale:

| Token | Value | Usage in this phase |
|-------|-------|----------------------|
| xs | 4px | Icon gaps, swatch check-mark padding |
| sm | 8px | Compact spacing inside accordion rows, sticky-bar internal gaps |
| md | 16px | Default card padding (`p-4`), Renkler card internal row gaps |
| lg | 24px | Section padding (`p-6`), spacing between the 3 decision-surface cards |
| xl | 32px | Column gap between preview and options (`gap-8`/`lg:gap-10`, existing — unchanged) |
| 2xl | 48px | Major section breaks (detail-page related-products spacing, unchanged) |
| 3xl | 64px | Page-level spacing (`/urunler` page padding, unchanged) |

Exceptions:
- Touch targets on interactive controls (color swatches, accordion summary rows, sticky-bar CTA) hold `min-h-11`/`min-h-12` (44/48px) — existing convention, applies to all new accordion `<summary>` rows and the mobile sticky bar.
- Color swatch buttons stay `h-12 w-12` (48px) — unchanged from `ColorPicker.tsx`, already meets touch-target minimums for both Taban and Kenar rows in the merged card.
- Mobile sticky bar has a fixed height band (`min-h-16`/64px content row) to guarantee price+CTA never compress below tap-target size; add `pb-[env(safe-area-inset-bottom)]` for iOS safe area.

## Typography

Existing scale from `globals.css`/Tailwind, reused as-is — no new sizes/weights introduced:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular, Instrument Sans) | 1.5 |
| Label / spec | 11–12px (`text-[11px]`/`text-xs`, uppercase tracked `spec-label`/`spec-value`) | 600–700 (semibold/bold) | 1.2 |

"Kenar (Overlok) Rengi") with one `h2` "Renkler" plus two sub-labels (see Copywriting Contract) —
sub-labels use the Label role, not a new heading size.

## Motion Contract (extends Phase 10 allowlist)

This phase's motion changes are prescriptive, not exploratory — they implement D-02, D-03, D-10 exactly:

| Surface | Remove | Keep |
|---------|--------|------|
| Stepper (`MatConfigurator.tsx`) | Animated fill bar (`motion.div` width spring), shine-sweep overlay, spring checkmark badge scale-in, `layout` prop on step badges | Static 3-segment indicator; CSS `transition-colors` for active/done text-color changes only |
| Preview panel | `AnimatePresence` spring scale transition (`scale: 0.97→1→1.02`, spring stiffness 200/damping 25) | A single ≤200ms opacity crossfade between preview states (D-03); implement via CSS `transition-opacity duration-200` or a minimal `framer-motion` `opacity`-only variant — no scale/spring |
| `ColorPicker` swatches | `whileHover={{ scale: 1.15, boxShadow }}`, `whileTap={{ scale: 0.92 }}`, spring checkmark rotate-in, `layoutId` glow ring, radial-gradient glow pulse | Simple `focus-visible` outline (existing) + CSS `:hover`/`:active` border-color transition; `btn-press` class if converted to a button-like pattern |
| Options column wrapper | `StaggeredReveal` (scroll-triggered stagger + blur-in per child) — remove entirely per D-10 | Render children directly in a plain `<div className="space-y-7">` |
| Extras accordion | N/A (new surface) | Native `<details>`/`<summary>` disclosure triangle rotation (existing pattern already used on `/urunler/[slug]` accordions, `PlusIcon` rotate-45 on open) — CSS-only, no framer-motion |
| Sticky bar | N/A (new surface) | Entrance: none required (D-06, always visible from load — no slide-in animation). Only `btn-press` on the CTA button |
| `ConfigSummary` "Eklendi" feedback | Out of scope this phase — `ConfigSummary`'s own `AnimatePresence`/spring badge is not listed in D-01–D-13; leave as-is unless it conflicts with the new sticky bar's simpler state (if duplicated logic is extracted to a shared hook, the animation must still respect `prefers-reduced-motion`) | — |

All retained motion (crossfade, `btn-press`, disclosure rotation, hover/focus transitions) MUST be
disabled or reduced to instant/no-op under `prefers-reduced-motion: reduce`, per the existing
`globals.css` pattern (see lines ~219, ~565, ~748) — this is a hard carry-over from Phase 10, not
optional for new code in this phase.
